Do nodes always need to restart after backend selection?

Toby Corkindale Wed, 03 Apr 2013 21:16:10 -0700 

Hi,
According to the docs at the following URL, it is necessary to reboot all Riak nodes after setting the bucket property for backend. 
This seems really drastic, and we'd like to avoid having to do this!
See:
http://docs.basho.com/riak/1.3.0/tutorials/choosing-a-backend/Multi/
I wondered if the restart of the whole cluster can be avoided? Perhaps we could set the bucket properties prior to setting any keys within it? 

Thanks in advance,
Toby
