On Mar 10, 2011, at 8:50 PM, Sean Cribbs wrote:


Antonio,
Sorry it's a little buried in the documentation, but be aware that Reduce phases *may* be called more than once (CouchDB calls this rereduce). What you experience in the last example is a demonstration of that: you must structure your reduce function such that it can receive as inputs anything that it outputs. The array/ list that you return as the output will be concatenated with more values on the subsequent applications of the function. Your reduce function can (almost) never assume that it is operating on the entire result set.
