HTTP Access Log/Performance Log issues

riakuser01 Thu, 25 Sep 2014 02:17:06 -0700 

Hi,

I am trying to use the HTTP Access logging on my Riak cluster to log the
request information, and I'm running into some issues. When the cluster
starts up, it begins logging HTTP requests as expected. However, at the end
of the current hour, these logs cease to be written to. I imagine it is
trying to roll the log file at the end of the hour, but failing.

I am currently using Riak 1.4.8, and have the following in my app.config
file:

 {webmachine, [{webmachine_logger_module, webmachine_logger},
              {log_handlers, [
                {webmachine_access_log_handler, ["/var/log/riak/access"]},
                {webmachine_perf_log_handler, ["/var/log/riak/access"]}
              ]}
           ]},
