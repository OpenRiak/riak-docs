Re: Using /etc/hosts entry and not ip in configuration files ..

Alex Moore Thu, 19 Sep 2013 17:01:13 -0700

Hey Mike,

Another option for your app.config would be to bind everything to "0.0.0.0", which will make Riak listen on all interfaces.  This combined with using a domain name in the vm.args should prevent you from having to do renames/changing ips in the config files.  
If you do this, please consider using a VPC to limit access to the machines.  

A good resource to read over is Amazon's Riak Whitepaper: http://media.amazonwebservices.com/AWS_NoSQL_Riak.pdf, as it goes through some operation considerations and tuning points that might be useful to you.

Thanks,
Alex
