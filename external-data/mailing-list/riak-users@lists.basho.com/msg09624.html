> Yeah sumo documentation could use some work :). You were
> right to use :vector-clock, and turns out it doesn't work
> because of a bug. The good news is I just fixed it [1] and
> pushed up a new release, 0.2.0. Readme is now updated and
> the release should be on Clojars already [2].
> 
> The rest of this explanation assumes you have {allow_mult,
> true} set as a bucket property. You can find a description
> of how to change that here [3]
> 
> The basic workflow with sumo (and likely any other Riak
> client) should be:
> 
> 1. GET values, which in sumo will return a seq of 0 or more
> results (0 corresponds with not found, more than one means a
> conflict)
> 2. If there are more than 1 result, resolve the conflicts
> (I'll expand on this later in the response)
> 3. Update the single, resolved value with whatever you were
> trying to do
> 4. Write back to Riak with the vector clock returned from
> step 1. (sumo will return a seq of riak objects, each of
> which have identical vclocks, so you it doesn't matter which
> one you choose).
> 
> Here's a gist that shows this more explicitly [4].
> 
> Now, how do you actually do the conflict resolution? The
> short story is it's 'app specific', but things like CRDTs
> [5] make it easier. I have a Clojure implementation of some
> basic
> CRDTs, called knockbox [6].
> 
> Hope this helps, and don't hesitate ask more.
