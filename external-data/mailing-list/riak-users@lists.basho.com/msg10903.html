Hi Mattias,

The following curl query simply counts the number of inputs, and has worked 
well for me in the past. Can you please run it against the cluster a couple of 
times and see if it also return varying number of results?

curl -XPOST http://localhost:8098/mapred 
  -H 'Content-Type: application/json' 
  -d '{"inputs":{
           "bucket":"som-bucket",
           "index":"userid_bin",
           "key":"18481123123"
       },
       "query":[{"reduce":{"language":"erlang",
                           "module":"riak_kv_mapreduce",
                           "function":"reduce_count_inputs"
                           }}]}'

Best regards,

Christian
