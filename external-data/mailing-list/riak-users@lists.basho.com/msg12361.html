Hey Darren,

As Dave said, the reason for a bad file error is the beam was compiled with one version of the Erlang VM and it is not being run in a different version. My suggestion would be to completely remove all RiakCS installations you have(from source and from package) and re-install from the .deb package. This has the correct version of Erlang packaged within it so you shouldn't have a problem.
