Ah thanks, I didn't know about the keys=false option.  I eventually
found the key streaming feature an built it into my rial wrapper.  The
weird problems with link headers went away.

On Monday, May 10, 2010, Sean Cribbs <s...@basho.com> wrote:
> Rick,
>
> If you don't want to list the keys when retrieving the bucket properties, 
> pass the keys=false query string.  When you have lots of keys, it's best to 
> use keys=stream to retrieve them, which will prevent Riak from building a 
> huge JSON object in memory. Instead, each chunk in the stream will be a JSON 
> object containing a number of keys.
