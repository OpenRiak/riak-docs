See Riak as a plain k/v store for saving, and later apply map/reduce
for further analysis.

> userId, partnerId, {products: [{productId: value, productName: value,
> productPrice: value, productUrl: value, productDescription: value},
> ...]}, orderId, date

For logging I would just make that a JSON object and save it.
Depending on your query patterns you would want to have a bucket per
partner, or a bucket for time slice. Search the mailing list, there
was a thread describing a similar use-case.

Francisco
