&gt;&gt; &gt;&gt; Hi Gavin,
&gt;&gt; &gt;&gt;
&gt;&gt; &gt;&gt; How have you configured your `storage\_schedule`? It's possible that
&gt;&gt; &gt;&gt; the job for calculating storage statistics hasn't occurred (or
&gt;&gt; &gt;&gt; finished) yet.
&gt;&gt; &gt;&gt;
&gt;&gt; &gt;&gt; You can manually trigger storage usage calculation by invoking the
&gt;&gt; &gt;&gt; following command:
&gt;&gt; &gt;&gt;
&gt;&gt; &gt;&gt; sudo riak-cs-storage batch
&gt;&gt; &gt;&gt;
&gt;&gt; &gt;&gt; From there, you can retrieve statistics from the `usage` bucket with
&gt;&gt; &gt;&gt; `s3cmd`:
&gt;&gt; &gt;&gt;
&gt;&gt; &gt;&gt; s3cmd get
&gt;&gt; &gt;&gt; s3://riak-cs/usage/.abj.20130501T000000Z.20130531T000000Z
&gt;&gt; &gt;&gt;
&gt;&gt; &gt;&gt; Also, here is a script I wrote up to access usage and storage
&gt;&gt; &gt;&gt; statistics with `s3cmd`:
&gt;&gt; &gt;&gt;
&gt;&gt; &gt;&gt; https://gist.github.com/hectcastro/126b5657f228096775c6
&gt;&gt; &gt;&gt;
