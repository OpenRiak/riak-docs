&gt;&gt; I am at the start of configuring a multi-node development
&gt;&gt; environment with Riak 1.4 on Ubuntu 12.04. Riak was installed
&gt;&gt; using apt-get.
&gt;&gt; 
&gt;&gt; I stopped the one node test of Riak with sudo riak stop but ps
&gt;&gt; -ef | grep erlang shows:
&gt;&gt; 
&gt;&gt; /usr/lib/riak/erts-5.9.1/bin/epdm -daemon (owner of the process
&gt;&gt; is riak)
&gt;&gt; 
&gt;&gt; after I ran the stop command.
&gt;&gt; 
&gt;&gt; Is this expected behaviour?
&gt;&gt; 
&gt;&gt; Should I use kill to stop the daemon?
&gt;&gt; 
&gt; 
&gt; No, you can leave epmd alone, it is started the first time a
&gt; distibuted erlang node is started, but will survive that node's
&gt; death. It is used to advertise what ports distributed erlang is
&gt; using on a machine, so erlang nodes can communicate.
