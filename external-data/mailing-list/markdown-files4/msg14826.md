2014-09-01 1:47 GMT+08:00 Eric Redmond :

&gt; Creating a search schema is non blocking, meaning that it will return ok
&gt; as it continues to process and propagate the schema on the backend.
&gt; Attempting to bind an index to a custom schema is generally because 1) the
&gt; schema failed to create (you can check out the solr.log or error.log for
&gt; details), 2) you're attempting to use the schema too fast, before it's had
&gt; time to propagate across the cluster.
&gt;
&gt; In your case, it looks like this first.
&gt;
&gt; Since you added.
&gt;
&gt; 
&gt;
&gt; You have to define a fieldtype named "ignored".
&gt;
&gt;  multiValued="true" class="solr.StrField" /&gt;
&gt;
&gt; Hope that helps,
&gt; Eric
