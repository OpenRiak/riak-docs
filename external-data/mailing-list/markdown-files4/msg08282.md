&gt;&gt;&gt; On 09.08.2012, at 22:47, Joseph Blomstedt  wrote:
&gt;&gt;&gt; 
&gt;&gt;&gt;&gt; Yes, this makes sense unfortunately. 'riak-admin transfers' isn't
&gt;&gt;&gt;&gt; going to work for you in a mixed 0.14.2 and 1.2 cluster.
&gt;&gt;&gt;&gt; 
&gt;&gt;&gt;&gt; Between 0.14.2 and 1.0, the entire cluster system was revamped. One
&gt;&gt;&gt;&gt; consequence of this change was that 'riak-admin transfers' would only
&gt;&gt;&gt;&gt; work on the 1.0+ nodes in the cluster, not any of the 0.14.2 nodes. At
&gt;&gt;&gt;&gt; the time, this wasn't a major issue because you could just use the
&gt;&gt;&gt;&gt; command on the right nodes and get the information you needed until
&gt;&gt;&gt;&gt; all nodes were eventually upgraded.
&gt;&gt;&gt;&gt; 
&gt;&gt;&gt;&gt; For Riak 1.2, 'riak-admin transfers' has been changed again. This
&gt;&gt;&gt;&gt; time, in a mixed cluster 'riak-admin transfers' only works on the
&gt;&gt;&gt;&gt; older nodes, not the Riak 1.2 nodes. For example, in a mixed 1.1 and
&gt;&gt;&gt;&gt; 1.2 cluster, you can only use riak-admin transfers on the 1.1 nodes
&gt;&gt;&gt;&gt; until all have been upgraded.
&gt;&gt;&gt;&gt; 
&gt;&gt;&gt;&gt; Unfortunately, the combination doesn't work out well for you in this
&gt;&gt;&gt;&gt; case. Riak 0.14.2 transfers fails if there are any 1.0+ nodes in the
&gt;&gt;&gt;&gt; cluster, and Riak 1.2 transfers fails if there are any &lt;1.2 nodes in
&gt;&gt;&gt;&gt; the cluster. Both are true, and therefore neither versions of Riak
&gt;&gt;&gt;&gt; can properly give you transfer information.
&gt;&gt;&gt;&gt; 
&gt;&gt;&gt;&gt; Of course, the lack of being able to monitor transfers doesn't mean
&gt;&gt;&gt;&gt; things aren't actually working. Running 'riak-admin member\_status' and
&gt;&gt;&gt;&gt; 'riak-admin ring\_status' on the newer nodes should provide enough
&gt;&gt;&gt;&gt; detail about what's going on to see if your cluster is moving along.
&gt;&gt;&gt;&gt;
