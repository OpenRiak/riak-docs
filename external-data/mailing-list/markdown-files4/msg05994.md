side-by-side, so auto-update shouldn't affect it in that regard. It could
be the delated ldconfig that happens after package install that errored our
or didn't run. Either way, glad things are working for you now.

&gt; Since we didn't build packages for Ubuntu 11.10 32bit, and instead just
&gt; did 64bit packages, I assume you are using the 32bit .deb from
&gt; http://basho.com/resources/downloads/ In that case, that .deb was
&gt; built using ubuntu 10.04 and you are probably running into a fairly common
&gt; issue with a mismatch in libssl libraries. Ubuntu 11.10 was the first
&gt; version that shipped without libssl version 0.9.8 which is needed by the
&gt; erlang vm that ships with Riak. Your fix should be as easy as:\*\*\*\*
&gt;
&gt; \*\* \*\*
&gt;
&gt; sudo apt-get install libssl0.9.8\*\*\*\*
&gt;
&gt; \*\* \*\*
&gt;
&gt; Install that and try starting Riak again. If you are still having
&gt; troubles, let us know.\*\*\*\*
