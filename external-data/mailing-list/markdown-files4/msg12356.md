Am 24.09.13 07:03, schrieb xsanch:
&gt; I had the same issue, when I straced the beam i saw a very high number of
&gt; read/write to entropy files, I just disabled the entropy in configuration
&gt; file app.config:
&gt; 
&gt; %% Enable active anti-entropy subsystem + optional debug
&gt; messages:
&gt; %% {anti\_entropy, {on|off, []}},
&gt; %% {anti\_entropy, {on|off, [debug]}},
&gt; {anti\_entropy, {off, []}},
&gt; 
&gt; 
&gt; Since then the process takes only about 2-4 percents cpu instead of 160. 

Interesting. Someone knows what this entropy thing is doing exactly? Can
it be switched off by default maybe?
