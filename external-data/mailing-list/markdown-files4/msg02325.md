On Tue, Feb 15, 2011 at 14:57, Dan Reverri  wrote:
&gt; Riak maintains a single value per key and provides mechanisms (vector
&gt; clocks) to detect/resolve conflicting values. In the proposed use case the
&gt; multiple copies would overwrite each other and Riak, by default, would
&gt; return a single value for a requested key.
&gt; Behind the scenes Riak determines the appropriate value per key using vector
&gt; clocks. More information about vector clocks is available here:
&gt; http://blog.basho.com/2010/01/29/why-vector-clocks-are-easy/
&gt; http://blog.basho.com/2010/04/05/why-vector-clocks-are-hard/
