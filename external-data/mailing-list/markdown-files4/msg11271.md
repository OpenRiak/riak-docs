Looks like the node was started with node name set in the vm.args to 
riak@127.0.0.1 and then restarted with a different name riak@10.147.29.57. If 
this is a single node that has not yet joined a cluster, simply stop the node 
with riak stop, delete the contents of riaks ring directory, and start the 
node.
