&gt;&gt;&gt;&gt; A few answers, thoughts and questions.
&gt;&gt;&gt;&gt;
&gt;&gt;&gt;&gt; 1. Each bucket allows (if after v1.1/1.2) replication. In 1.1 and above
&gt;&gt;&gt;&gt; there is a repl value that accepts a true or false value. True turns on
&gt;&gt;&gt;&gt; realtime and fullsync abilities. In 1.2 above has additional boolean
&gt;&gt;&gt;&gt; parameters of realtime, fullsync, or both. Enabling the property via
&gt;&gt;&gt;&gt; command line:
&gt;&gt;&gt;&gt;
&gt;&gt;&gt;&gt; curl -v -XPUT -H "Content-Type: application/json" \
&gt;&gt;&gt;&gt; -d '{"props":{"repl":true}}' \
&gt;&gt;&gt;&gt; http://127.0.0.1:8091/riak/my\_bucket
&gt;&gt;&gt;&gt;
&gt;&gt;&gt;&gt; 2. Running both styles of replication should be fine. They're on by
&gt;&gt;&gt;&gt; default to start. In the particular situation you describe - using realtime
&gt;&gt;&gt;&gt; on all the time should work well and then only using the fullsync when the
&gt;&gt;&gt;&gt; ship docks and connects at a higher speed - using something to trigger 
&gt;&gt;&gt;&gt; that.
&gt;&gt;&gt;&gt;
&gt;&gt;&gt;&gt; A few additional questions:
&gt;&gt;&gt;&gt;
&gt;&gt;&gt;&gt; - I recall we spoke about data sizes of 3-5k per ship, but then
&gt;&gt;&gt;&gt; there was all of the data that would go along with each client, could 
&gt;&gt;&gt;&gt; you
&gt;&gt;&gt;&gt; provide more elaboration around what count, sizes, connections to other
&gt;&gt;&gt;&gt; elements and related information data? Why type/pieces would need to go 
&gt;&gt;&gt;&gt; to
&gt;&gt;&gt;&gt; each ship, etc.
&gt;&gt;&gt;&gt; - For the connections to each ship during satellite link what does
&gt;&gt;&gt;&gt; the bandwidth, latency and other characteristics look like? Latency 
&gt;&gt;&gt;&gt; times
&gt;&gt;&gt;&gt; of 800, 1500, 6000 or possibly higher 8000, 10000?
&gt;&gt;&gt;&gt;
&gt;&gt;&gt;&gt; Another Architectural Thought:
&gt;&gt;&gt;&gt;
&gt;&gt;&gt;&gt; - One idea that stands out would be to use Riak as the primary
&gt;&gt;&gt;&gt; cluster but to implement a client that does replication itself 
&gt;&gt;&gt;&gt; specifically
&gt;&gt;&gt;&gt; for a bucket (or buckets). It seems like, from my understanding so far,
&gt;&gt;&gt;&gt; that the client might be the key mechanism to control any type of
&gt;&gt;&gt;&gt; replication - with or without MDC being used. Basically following a
&gt;&gt;&gt;&gt; standard hub-and-spoke server & client model.
