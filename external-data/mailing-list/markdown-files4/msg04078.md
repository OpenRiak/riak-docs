On Mon, Jul 25, 2011 at 9:40 AM, francisco treacy &lt;
francisco.tre...@gmail.com&gt; wrote:

&gt; It's awesome for ad-hoc querying, at least. An example can better explain.
&gt;
&gt; Consider this:
&gt;
&gt; db.add('users').map('query', '.address .street where
&gt; .weight:expr(x&lt;180), .exempt && .acl .state:val("1101") &&
&gt; !.expired').run()
&gt;
&gt; as opposed to:
&gt;
&gt; db.add('users').map(function(v) {
&gt; v = Riak.mapValuesJson(v)[0];
&gt; var result = [];
&gt; if ((v.weight &lt; 180 || v.exempt) && v.acl && v.acl.state === '1101'
&gt; && !v.expired) {
&gt; if (v.address) {
&gt; result.push(v.address.street);
&gt; }
&gt; }
&gt; return result;
&gt; }).run()
&gt;
&gt;
&gt; riak-ql is basically adding some query sugar (where, &&) on top of
&gt; JSONSelect... which you can try it out here:
&gt; http://jsonselect.org/#tryit
&gt;
