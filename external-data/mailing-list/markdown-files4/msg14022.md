&gt; On 6 April 2014 15:02, Matthew Von-Maszewski  wrote:
&gt;
&gt;&gt; Edgar,
&gt;&gt;
&gt;&gt; In Riak 1.4, there is no advantage to using empty values versus deleting.
&gt;&gt;
&gt;&gt; leveldb is a "write once" data store. New data for a given key never
&gt;&gt; physically overwrites old data for the same key. New data "hides" the old
&gt;&gt; data by being in a lower level, and therefore picked first.
&gt;&gt;
&gt;&gt; leveldb's compaction operation will remove older key/value pairs only
&gt;&gt; when the newer key/value is pair is part of a compaction involving both new
&gt;&gt; and old. The new and the old key/value pairs must have migrated to
&gt;&gt; adjacent levels through normal compaction operations before leveldb will
&gt;&gt; see them in the same compaction. The migration could take days, weeks, or
&gt;&gt; even months depending upon the size of your entire dataset and the rate of
&gt;&gt; incoming write operations.
&gt;&gt;
&gt;&gt; leveldb's "delete" object is exactly the same as your empty JSON object.
&gt;&gt; The delete object simply has one more flag set that allows it to also be
&gt;&gt; removed if and only if there is no chance for an identical key to exist on
&gt;&gt; a higher level.
&gt;&gt;
&gt;&gt; I apologize that I cannot give you a more useful answer. 2.0 is on the
&gt;&gt; horizon.
&gt;&gt;
&gt;&gt; Matthew
