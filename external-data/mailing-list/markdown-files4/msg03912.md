Because it's an accessor method, that would need to be

self.bucket\_name = 'tcweb\_roots'

On Fri, Jul 1, 2011 at 1:18 PM, Aphyr  wrote:

&gt; class TcWeb::Root
&gt; include Ripple::Document
&gt; bucket\_name = 'roots' # or tcweb\_roots, whatever
&gt; ...
&gt; end
&gt;
&gt;
&gt; On 07/01/2011 08:25 AM, Thomas Fee wrote:
&gt;
&gt;&gt; I'm currently using Ripple with the application name prepended to the
&gt;&gt; typename in an effort to artificially create a namespace for app, to not
&gt;&gt; collide with other apps, e.g.
&gt;&gt;
&gt;&gt; class TcwebRoot
&gt;&gt; include Ripple::Document
&gt;&gt; property :typed\_root\_symbol, String, :presence =&gt; true
&gt;&gt; key\_on :typed\_root\_symbol
&gt;&gt; # et cetera
&gt;&gt; end
&gt;&gt;
&gt;&gt; Where "Tcweb" is the appname functioning as a namespace prefix. The
&gt;&gt; object class should ideally be called just "Root". I would prefer to not
&gt;&gt; have "Tcweb" mangled into the classname.
&gt;&gt;
&gt;&gt; Does Ripple allow this sort of thing?...
&gt;&gt;
&gt;&gt; Ripple::namespace("Tcweb")
&gt;&gt; class Root
&gt;&gt; include Ripple::Document
&gt;&gt; property :typed\_root\_symbol, String, :presence =&gt; true
&gt;&gt; # et cetera
&gt;&gt; end
&gt;&gt;
&gt;&gt; Note:
&gt;&gt; With my name mangling, plus Ripple's name mapping conventions, a
&gt;&gt; TcwebRoot object is currently queried like this...
&gt;&gt; http://172.22.59.51:8098/riak/\*\*tcweb\_roots/%24200-KOR%7C0
