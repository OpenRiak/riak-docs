&gt;&gt;&gt; Following up on Alex's comments -
&gt;&gt;&gt;
&gt;&gt;&gt; If you know which bytes you need to slice, you can store this in a
&gt;&gt;&gt; secondary index. You can perform range queries across secondary indices (as
&gt;&gt;&gt; well as keys).
&gt;&gt;&gt;
&gt;&gt;&gt; As long as you're storing your data in a way that allows it to be read
&gt;&gt;&gt; by either Erlang or JavaScript, you should be able to query over it in
&gt;&gt;&gt; MapReduce. This is typically regarded as a Bad Ideaâ„¢ since an MR query will
&gt;&gt;&gt; need to scan all keys in a bucket (which effectively means scanning the
&gt;&gt;&gt; entire cluster) and is best done as an infrequent activity to transform
&gt;&gt;&gt; data.
