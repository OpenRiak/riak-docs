On Thu, Mar 21, 2013 at 08:36:48AM -0600, Jared Morrow wrote:
&gt; Yes, this is true. If you would like to make your own package for 32bit,
&gt; you can build from source and do a 'make package', but we highly recommend

Just in case you'd would like to make own package, add
'RELEASE=' to 'make package' command, e.g.

 $ make package RELEASE=1

Otherwise it'd bail out with an error message.
