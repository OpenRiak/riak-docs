&gt;&gt; Hi Allen,

&gt;&gt; When generating a release package with rebar the intention is to create a
&gt;&gt; fully independent release package that runs without any external
&gt;&gt; dependencies. During the release generation rebar will analyze the
&gt;&gt; specification of an application to determine dependencies. The tool will
&gt;&gt; then collect all the dependencies (including those from the system level
&gt;&gt; Erlang installation) and package them into a release. Tsung is not part of
&gt;&gt; the riak dependency specification and should not be pulled into the release.
&gt;&gt; My guess is that macports has configured Erlang in such a way that the Tsung
&gt;&gt; library is included in all releases. We have to look into this a bit further
&gt;&gt; and try to reproduce the issue locally.
