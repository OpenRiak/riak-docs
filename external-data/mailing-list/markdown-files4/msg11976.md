Andre Lohmann  wrote:

al&gt; When Hitting Server 1 again for the same file, will it be fetched
al&gt; again piece by piece from the other servers, or is there some kind
al&gt; of caching (prefered in memory) available, to deliver this file from
al&gt; cache the next time?

Riak CS does not have such a caching feature at this time. Today, the
cache resides in the OS page cache of the Riak servers. Adding a
feature to cache extremely hot objects is intended but has not yet been
scheduled on Basho's roadmap.
