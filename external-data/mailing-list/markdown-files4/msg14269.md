On Wed, May 28, 2014 at 2:36 AM, Jordan West  wrote:

&gt; Gavin,
&gt;
&gt; Are you able to stop all other traffic to Riak? Testing locally, the
&gt; crashes only start occurring once the first request to a key in the
&gt; mis-configured bucket is issued after restarting. Using curl I am able to
&gt; change the property back to 3, as an integer, as long as no keys are
&gt; requested in that bucket after restarting isn't made.
&gt;
&gt; Otherwise, if you can keep Riak running long enough, executing the
&gt; following from `riak attach` should have the same end result:
&gt;
&gt; riak\_core\_bucket:set\_bucket(&lt;&lt;"b"&gt;&gt;, lists:keystore(n\_val, 1,
&gt; riak\_core\_bucket:get\_bucket(&lt;&lt;"b"&gt;&gt;), {n\_val, 3})).
&gt;
&gt; A third option, I haven't tried, is to temporarily disable active
&gt; anti-entropy (the subsystem where the error seems to be coming from), until
&gt; you've had a chance to make the fix.
&gt;
&gt; Jordan
