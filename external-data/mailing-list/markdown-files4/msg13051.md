&gt;&gt;&gt;&gt; Strong consistency:
&gt;&gt;&gt;&gt; How does one activate this? As i understand, either the pb or the http 
&gt;&gt;&gt;&gt; clients do not accept the consistent property.
&gt;&gt;&gt;&gt; 
&gt;&gt;&gt;&gt; 
&gt;&gt;&gt;&gt; Using strong consistency in the Tech Preview is a two-step process. First, 
&gt;&gt;&gt;&gt; turn on the feature in your riak.conf (the following command assumes you 
&gt;&gt;&gt;&gt; are using a devrel):
&gt;&gt;&gt;&gt; 
&gt;&gt;&gt;&gt; for d in dev/dev\*; do sed -e 's/## enable\_consensus = 
&gt;&gt;&gt;&gt; true/enable\_consensus = true/' -i $d/etc/riak.conf; done
&gt;&gt;&gt;&gt; 
&gt;&gt;&gt;&gt; Then re(start) the cluster.
&gt;&gt;&gt;&gt; 
&gt;&gt;&gt;&gt; With the enable\_consensus=true un-commented and the cluster running create 
&gt;&gt;&gt;&gt; a strongly consistent bucket type:
&gt;&gt;&gt;&gt; 
&gt;&gt;&gt;&gt; riak-admin bucket-type create  '{"props": {"consistent": true}}'
&gt;&gt;&gt;&gt; riak-admin bucket-type activate 
&gt;&gt;&gt;&gt; 
&gt;&gt;&gt;&gt; You can create and activate the bucket type without having 
&gt;&gt;&gt;&gt; enabled\_consensus=true but both must be done to work with strongly 
&gt;&gt;&gt;&gt; consistent data.
&gt;&gt;&gt;&gt; 
&gt;&gt;&gt;&gt; Once that is done you can make requests, just like you would to eventually 
&gt;&gt;&gt;&gt; consistent data in Riak, but the semantics will be different. For example, 
&gt;&gt;&gt;&gt; if you make two PUT requests to the same key, w/ no vector clock\*, the 
&gt;&gt;&gt;&gt; second request will fail because the data already exists. More details on 
&gt;&gt;&gt;&gt; semantics of strongly consistent requests can be found in this PR: 
&gt;&gt;&gt;&gt; https://github.com/basho/riak\_kv/pull/710
&gt;&gt;&gt;&gt; 
&gt;&gt;&gt;&gt; Hope that helps get you started,
&gt;&gt;&gt;&gt; 
&gt;&gt;&gt;&gt; Jordan
&gt;&gt;&gt;&gt; 
&gt;&gt;&gt;&gt; \* in strong consistency, the "vector clock" is not really a vector clock. 
&gt;&gt;&gt;&gt; its called such for familiarity, but like any real vector clock exposed to 
&gt;&gt;&gt;&gt; the client, should be treated like an opaque context.
