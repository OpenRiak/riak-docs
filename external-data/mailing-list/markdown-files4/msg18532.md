&gt;&gt; Hi Guido,

&gt;&gt; You should attempt to change the bucket property ‘notfound\_ok’ from the 
&gt;&gt; default of ‘true' to ‘false'.
&gt;&gt; 
&gt;&gt; I.e 
&gt;&gt; 
&gt;&gt; curl -XPUT 127.0.0.1:10018/buckets/foo/props -H "Content-Type: 
&gt;&gt; application/json" -d '{"props":{"notfound\_ok": false}}'
&gt;&gt; 
&gt;&gt; This makes GET operations for non-existent keys slower as it forces an 
&gt;&gt; internal GET for each of the three copies.
&gt;&gt; 
&gt;&gt; https://docs.basho.com/riak/kv/2.1.1/developing/app-guide/replication-properties/#the-implications-of-notfound-ok
&gt;&gt; 
&gt;&gt; From what you describe, it sounds like only a single copy (out of the 
&gt;&gt; original three), somehow remain present in your cluster.
