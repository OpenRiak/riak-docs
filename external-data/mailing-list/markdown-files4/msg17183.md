Hi Shailesh,

I hadn't heard of JugglingDB. Looks interesting. There is no riak adapter
for it as far as I know. There is, however, a riak client library for
nodejs[0]. So I imagine an adapter could be written. Looking through the
examples in the read me it looks like solr would have to be used to
implement certain features.

How many folks out there would like to use something like juggling or
waterline?

-Alexander


[0] https://github.com/basho/riak-nodejs-client

&gt;&gt; Hello All!

&gt;&gt; Here’s the latest summary of the Riak users mailing list, latest client
&gt;&gt; releases and job openings at Basho.
&gt;&gt;
&gt;&gt; ## Latest Releases
&gt;&gt;
&gt;&gt;
&gt;&gt; - NodeJS Client 2.1.2 - includes improvements to closed connections
&gt;&gt; handling such as with AWS ELB. Release notes [1] and code [2].
&gt;&gt; - Java Client 2.0.5 - brings a number of fixes including a fix to a
&gt;&gt; DNS refresh issue and enhanced testing. Release notes [3] and maven repo
&gt;&gt; [4].
&gt;&gt;
&gt;&gt;
&gt;&gt; ## Recently Answered
&gt;&gt;
&gt;&gt;
&gt;&gt; - Magnus and Russell help Oleksiy clear up some confusion about the
&gt;&gt; hidden $bucket index [5] - but only when using leveldb. Nevertheless,
&gt;&gt; perhaps a $bucket query should not be accepted against a bitcask backend
&gt;&gt; [6].
&gt;&gt; - Robert notes that when joining nodes to clusters they must have the
&gt;&gt; same ring sizes [7].
&gt;&gt; - Agung is interested in working with riak\_ensemble [8][9] and has a
&gt;&gt; few questions about it. Carlos recommends taking a look at nkbase [10],
&gt;&gt; which uses riak\_ensemble, and Sargun goes into more detail [11].
&gt;&gt;
&gt;&gt;
&gt;&gt; ## Open Discussions
&gt;&gt;
&gt;&gt;
&gt;&gt; - Fred and Magnus are working with Oleksiy to determine the root
&gt;&gt; cause of inconsistent Solr search results [12].
&gt;&gt; - Bartek is looking for assistance on how to feed a mapreduce job
&gt;&gt; with the results of a full text search [13]
&gt;&gt; - Kinson is looking for guidance on implementing Basho Bench [14]
&gt;&gt; - There is an open question regarding whether it’s possible to
&gt;&gt; rebalance Riak KV without adding/removing nodes or making any hardware
&gt;&gt; changes [15]
&gt;&gt; - Luke thinks that Gerhard’s proposed architecture will ensure
&gt;&gt; replicas are distributed to every node, but invites someone with more
&gt;&gt; knowledge of the hashing method to chime in [16]
&gt;&gt; - Robert is looking for guidance on choosing EC2 instance types for
&gt;&gt; use with Riak/LevelDB [17]
&gt;&gt; - Vitaly is helping troubleshoot timeouts in a deployment of Riak KV
&gt;&gt; running on a cluster with 512 partitions [18]
&gt;&gt; - There’s an open question about debugging a post commit hook for a
&gt;&gt; bucket type in Riak 2.1.3 [19]
&gt;&gt; - Christopher explained that when configuring multiple Riak clients
&gt;&gt; in a cluster, you need to have each client use a different name. Chathuri
&gt;&gt; has some follow-up questions about the configuration [20]
&gt;&gt; - Vitaly and Chris are helping troubleshoot testing in a single node
&gt;&gt; implementation of Riak KV [21]
&gt;&gt;
&gt;&gt;
