Ripple/riak-client doesn't split a bunch of links into multiple
headers for you, right?

I'm trying the new associations feature which works great btw, but if
i'm limited to 100/150 links or a horrible workaround (in terms of
client code), I'd probably be better off storing that in an associated
document and resorting to map-reduce.

Something like (in Javascript):

var map = function(v) {
 var doc = Riak.mapValuesJson(v)[0];
 return doc.following.map(function(key) { return ['users', key] })
}

db
 .map(map)
 // [ eventually a .link({}) here ]
 .map("Riak.mapValuesJson")
 .run([['user\_relations', 'user1']])()

This approach also makes more sense to me now, it allows me to
decouple relationships from the users docs, leaving those lightweight
for other operations.

Thanks,
Francisco

&gt;&gt; Hi all,

&gt;&gt; We're developing social features in our app that involve Twitter-like
&gt;&gt; relations between users.
&gt;&gt;
&gt;&gt; As we have a 'users' bucket, the initial implementation is to add
&gt;&gt; links to following users (and link followers back) in order to create
&gt;&gt; a two-way relationship.
&gt;&gt;
&gt;&gt; User A  ==== following ====&gt; User B
&gt;&gt; User B ==== follower ====&gt; User A
&gt;&gt;
&gt;&gt; This makes sense and turns out to work perfectly fine, but I am
&gt;&gt; concerned with hitting limits rather soon. For some reason (webmachine
&gt;&gt; related?) there is an 8k limit for the Link header. I verified this
&gt;&gt; (documents with 8192b+ Link headers won't be persisted).
&gt;&gt;
&gt;&gt; Considering average key sizes and the typical usecase of our app, this
&gt;&gt; means we can store somewhere between 100 and 150 links - max. Given we
&gt;&gt; store different types of links (not only to other users), it'll
&gt;&gt; definitely won't work for us.
&gt;&gt;
&gt;&gt; So I guess my question is two-fold:
&gt;&gt; (a) is this limit intended? if not, how can it be eluded?
&gt;&gt; (b) are there better ways of modeling this scenario?
&gt;&gt;
&gt;&gt; I was thinking along the lines of storing this 'followers/following'
&gt;&gt; data in an associated document instead, and feeding a link phase keys
&gt;&gt; coming from a map function.
&gt;&gt;
&gt;&gt; Those of you who tried implementing Twitter in Riak :) ...how did you
&gt;&gt; approach the problem?
