&gt;&gt; The reduce\_identity function expects a two element tuple or a two element
&gt;&gt; list; you can see this by looking at the pattern matching in the function:
&gt;&gt;
&gt;&gt; https://github.com/basho/riak\_kv/blob/master/src/riak\_kv\_mapreduce.erl#L141
&gt;&gt;
&gt;&gt; A link phase returns a three element list (["bucket", "key", "tag"]). The
&gt;&gt; reduce\_identity function should probably be updated to handle the output of
&gt;&gt; a phase. In the mean time you can use the reduce\_set\_union function instead:
&gt;&gt; {
&gt;&gt; "inputs":{
&gt;&gt; "bucket":"bucket",
&gt;&gt; "key\_filters":[["neq", "foo"]]
&gt;&gt; },
&gt;&gt; "query":[
&gt;&gt; {"link":{}},
&gt;&gt; {"reduce":{"language":"erlang","module":"riak\_kv\_mapreduce",
&gt;&gt; "function":"reduce\_set\_union"}}
&gt;&gt; ]
&gt;&gt; }
