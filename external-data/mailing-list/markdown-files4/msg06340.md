2012/1/20 Sean Cribbs :
&gt; Be aware that the client quorums are only a restriction on what the client
&gt; will wait for. It is entirely possible for a write to succeed in the larger
&gt; case, but not meet the W/DW quorum requirements. That is, the request may
&gt; "fail" but the write may still occur.
&gt;
&gt;
&gt; On Fri, Jan 20, 2012 at 10:01 AM, Dmitry Demeshchuk 
&gt; wrote:
&gt;&gt;
&gt;&gt; Generally, using eventually consistent databases for e-commerce sounds
&gt;&gt; too risky.
&gt;&gt;
&gt;&gt; But I know that there was some e-commerce stealth startup using Riak
&gt;&gt; for their needs (probably not for all the data though, I don't know
&gt;&gt; any details). Amazon uses Dynamo, which is quite similar to Riak. So
&gt;&gt; NoSQL can be used in this niche somehow.
&gt;&gt;
&gt;&gt; Also, intuitively, most of the consistency problems might be avoided
&gt;&gt; by setting all w/dw values to maximum (better set r to maximum too, of
&gt;&gt; course).
&gt;&gt;
&gt;&gt; The hardest task I see here is to organize transactions for the cases
&gt;&gt; like "we remove a product from database and we should alter all the
&gt;&gt; orders that contain this product".
