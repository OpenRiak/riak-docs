Creating a search schema is non blocking, meaning that it will return ok as it 
continues to process and propagate the schema on the backend. Attempting to 
bind an index to a custom schema is generally because 1) the schema failed to 
create (you can check out the solr.log or error.log for details), 2) you're 
attempting to use the schema too fast, before it's had time to propagate across 
the cluster.

In your case, it looks like this first.

Since you added.



You have to define a fieldtype named "ignored".



Hope that helps,
