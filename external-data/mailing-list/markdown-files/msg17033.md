Here is a summary of what's come over our user list since the last recap.

## Announcements

\* John Musser, ProgrammableWeb founder and API expert, has joined Basho as
Vice President of Technology. Press release [1] The New Stack article [2]

## Latest Releases

\* Riak Go Client updates

- v1.5.1 released on February 11 — This is a maintenance release. Release
notes [3] Source code [4]

- v1.6.0 released on February 19 — This release includes fixes and a few
enhancements. Release notes [5] Source code [6]

## Recently Answered

\* Zeeshawn explained that Riak search operates via Distributed Solr in
response to a question about Solr join queries [7]

\* Vitaly and Zeeshan offered assistance on a question about whether Riak
Search indexes documents sent as JSON, and Colin wrote up a good summary of
the resolution to his issue [8]

\* Zeeshan helped resolve Riak Search questions where a record is deleted
and new one created. This is solved by Configuring the delete\_mode to
immediate. [9]

\* Christopher and Damien explained how to to set the epmd port listener via
riak.conf [10]

## Open Discussions

\* Alex answered a question about how to use mapreduce to run a monthly
report on daily and weekly data kept in different buckets [11]

\* Kinson is looking for guidance on implementing Basho Bench [12]

\* There is an open question regarding whether it’s possible to rebalance
Riak without adding/removing nodes or making any hardware changes [13]

\* Luke thinks that Gerhard’s proposed architecture will ensure replicas are
distributed to every node, but invites someone with more knowledge of the
hashing method to chime in [14]

\* Robert is looking for guidance on choosing EC2 instance types for use
with Riak/levelDB [15]

\* There is an open question about adding request handlers to enable
language detection in Solr for Riak nodes [16]

## Jobs at Basho

We have an opening for a Developer Advocate in our London office. [17]

## Weekend Reads

\* The Stack interviewed Basho’s European head of operations, Manu Marchal,
for their article “Why NoSQL is the Future of IoT” [18]

Have a great weekend,
