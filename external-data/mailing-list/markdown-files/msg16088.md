Here's your regular edition of The Recap.

## Code releases
\* a new PHP client was released since we last Recapped [1]

## Recently answered
\* John Daily helped a new Riak CS user customize his advanced.config
file, which is now in documentation [2]
\* Luke Bakken confirmed you can chain indexes together using the
Node.js client [3]
\* Multiple people help Gustavo determine it's best to write
middleware, not expose Riak to the public [4]
\* Kota confirms that 48/64GB RAM with 20TB of storage is typical for
Shankar's use case of Riak CS [5]
\* Andrew Stone helps discuss an ensembles failure scenario [6]
\* Steve Garon teaches us all to make Solr searches faster by
commenting out  [7]
\* Alexander gives a clear explanation of why Riak uniformly uses disk
space across a cluster [8]

## Still unanswered
\* Riak Search results are still trouble for this user [9]
\* Trenton has a concern with DNS caching in the Java client and would
like to discuss it [10]
\* Rustem sees different results from the same map/reduce query over cURL [11]
\* Amit needs some help getting Riak CS running [12]
\* A local java environment is throwing exceptions for Henning [13]

## Community events coming up
\* NYC Riak is meeting on 5/14 to discuss Bit.ly's real-time bidding
infrastructure [14]

## Recent presentations
\* Our friends at Uber presented on an open source project called
Ringpop: Presentation [15] & video [16]
\* I gave two high-level presentations at Big Data TechCon this week [17][18]
