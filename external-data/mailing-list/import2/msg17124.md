Hi All,

I've got a cluster in 2.1.1 and was wanting to join a 2.1.3 node to it

I am getting the following error:

# ./bin/riak-admin cluster join a-backend@10.0.0.199
Failed: a-backend@10.0.0.199 has a different ring\_creation\_size

So... for the new 2.1.3
# ./bin/riak-admin status | grep ring
ring\_creation\_size : 256
ring\_members : ['node-template@10.0.0.54']
\*ring\_num\_partitions : 256\*
ring\_ownership : <<"[{'node-template@10.0.0.54',256}]">>
rings\_reconciled : 0
rings\_reconciled\_total : 0

AND for node a-backend@10.0.0.199
# ./bin/riak-admin status | grep ring
ring\_creation\_size : 256
ring\_members : ['a-backend@10.0.0.199','b-backend@10.0.0.200',
\*ring\_num\_partitions : 64\*
ring\_ownership : <<"[{'b-backend@10.0.0.200',13},\n 
{'c-backend@10.0.0.201',13},\n {'d-backend@10.0.0.202',13},\n 
{'e-backend@10.0.0.203',12},\n {'a-backend@10.0.0.199',13}]">>

rings\_reconciled : 0
rings\_reconciled\_total : 32

The 'ring\_num\_partitions' are different which leaves me to believe this 
is the cause.


A.) How do I increase the 'ring\_num\_partitions' to 256 in the 2.1.1 
cluster \*\*OR\*\*

B.) How do I decrease the 'ring\_num\_partitions' to 64 in the 2.1.3 node?

Any ideas? recommendations?


Thanks in advance,
