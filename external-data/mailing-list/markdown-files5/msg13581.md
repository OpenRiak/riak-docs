&gt; Satya,
&gt;
&gt; In general, you always want to be querying Riak using a specific key value
&gt; to retrieve or update a specific value. Following this guideline, method
&gt; one is the correct data design. Please ensure that the default bucket
&gt; properties apply to all of these buckets as you do not want to have any
&gt; custom properties for such a large number of buckets.
&gt;
&gt; Method two will result in slow performance and most likely an object size
&gt; exceeding the recommended 1MB value.
&gt;
&gt; --
&gt; Luke Bakken
&gt; CSE
&gt; lbak...@basho.com

&gt; On Tue, Feb 4, 2014 at 4:58 AM, Satya Ranjan wrote:
&gt;
&gt;&gt; Hi Riak experts,
&gt;&gt;
&gt;&gt; I am designing a simple message box feature for a web application using
&gt;&gt; riak where an user can send messages to other registered users. So each
&gt;&gt; user will have a Inbox and Outbox just like an email system.
&gt;&gt;
&gt;&gt; Also users can search messages in their message box. Here I am providing
&gt;&gt; two different data model design. Could you please let me know if it makes
&gt;&gt; sense.
&gt;&gt;
&gt;&gt; 1) Each user will have a separate bucket for their messages. So user u1
&gt;&gt; will have a bucket u1\\_messages and u2 will have u2\\_messages. All messages
&gt;&gt; for an user will be added to his/her bucket. This will simplify search
&gt;&gt; operation as the search operation will only operate on user's bucket.
&gt;&gt;
&gt;&gt; With this approach there will be lot many buckets in riak. So for 10000
&gt;&gt; registered users, there will be 10000 buckets.
&gt;&gt;
&gt;&gt; So does this approach makes sense ? Is there any limitation or drawback
&gt;&gt; when there are too many buckets in riak ?
&gt;&gt;
&gt;&gt; 2) Common message bucket for all the users. The messages in the bucket
&gt;&gt; will be tagged with the user id. So one big bucket for all the messages.
&gt;&gt; Search operation will operate on that big bucket.
&gt;&gt;
&gt;&gt; Please suggest which of the above approach makes sense. I would really
&gt;&gt; appreciate any help on this.
