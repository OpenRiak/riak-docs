On Oct 6, 2013 4:35 PM, "Alex Rice"  wrote:
&gt;
&gt; Hi all, unable to find these in the Riak docs
&gt;
&gt; 1) Can someone point me at directions for cleanup of orphaned Links
&gt; after an object is Deleted from Riak. Post-commit hooks or something?
&gt; I really hope I dont have to learn Erlang just to clean up bad links
&gt; after deleting objects!

There aren't any docs for that particular case. In fact, this is yet
another example of why we no longer recommend using Links at all.

If you're just beginning to design a production Riak model I'd recommend
investigating alternative models that can leverage direct key lookups, or
2i, or yokozuna.

&gt; 2) Are links stored in key space or value space, or somewhere else?
&gt; (thinking saving memory with Bitcask backend) When viewing links with
&gt; the rekon tool, the paths look a bit verbose, prefixed with "/bucket"
&gt; etc. So I was just wondering if they use more memory with bitcask.

Link data is metadata, so they're stored along with the value.
