&gt;&gt; From the 1.4.8 release notes:
&gt;&gt;
&gt;&gt; IMPORTANT We recommend removing current AAE trees before upgrading. That
&gt;&gt; is, all files under the anti\\_entropy sub-directory. This will avoid
&gt;&gt; potentially large amounts of repair activity once correct hashes start
&gt;&gt; being added. The data in the current trees can only be fixed by a full
&gt;&gt; rebuild, so this repair activity is wasteful. Trees will start to build
&gt;&gt; once AAE is re-enabled. To minimize the impact of this, we recommend
&gt;&gt; upgrading during a period of low activity.
