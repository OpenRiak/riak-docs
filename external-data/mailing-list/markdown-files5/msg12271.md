On Mon, Sep 16, 2013 at 12:16 AM, Jeremiah Peschka &lt;
jeremiah.pesc...@gmail.com&gt; wrote:

&gt; Welp, here's what I did and it seemed to work for me:
&gt;
&gt; Created a new VPC
&gt; Created a subnet in said VPC
&gt; Created a default route for SSH connectivity
&gt; Created a security group with the TCP/IP settings outline in the AWS PDF
&gt; Created two instances using the Riak AMI, both inside the VPC.
&gt;
&gt; On each instance, I edited app.config to listen on the local IP address
&gt; (found using curl http://169.254.169.254/latest/meta-data/local-ipv4)
&gt;
&gt; On the first instance, I started riak and was able to ping the node
&gt; successfully from the local host and from instance two.
&gt;
&gt; On the second instance I started riak and then ran the following -
&gt; http://pastebin.com/CaHH3Eve
&gt;
&gt; Each node's vm.args is using the IP address for the `-name` parameter.
&gt;
&gt; If you're in a VPC, you may want to check the box for "Enable DNS
&gt; hostname support for instances launched in this VPC."
