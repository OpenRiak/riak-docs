On Fri, Sep 26, 2014 at 4:51 AM, Anton Khalikov  wrote:
&gt; Hello Basho guys
&gt;
&gt; I think we have found a small bug in riak/riak-cs debian init scripts
&gt; (/etc/init.d/riak{,-cs}). When used in CRON, ie for restarting riak process
&gt; on a schedule, init scripts fail because they don't set PATH variable to
&gt; recommended value. Without setting proper PATH, init script is unable to
&gt; find start-stop-daemon.
&gt;
&gt; Recommended value for PATH for debian init scripts can be found in
&gt; /etc/init.d/ssh and many other debian init scripts and looks like:
&gt;
&gt; export PATH="${PATH:+$PATH:}/usr/sbin:/sbin"
&gt; or
&gt; PATH=/sbin:/bin:/usr/sbin:/usr/bin
&gt;
