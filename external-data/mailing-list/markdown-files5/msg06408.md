Also note that on 1.0 and later you can set the "search" bucket property to
true and that will install the hook for you.

On Wed, Jan 25, 2012 at 9:51 AM, charlesugwuh wrote:

&gt; I figured this out.
&gt;
&gt; So I'm using the https://github.com/basho/riak-php-client PHP Riak
&gt; Client .
&gt; I was using the setProperties method to set the commit hooks for the bucket
&gt; to enable search.
&gt;
&gt; However, I was defining the bucket properties like so: '$prop\\_arr =
&gt; array('precommit' =&gt; array('mod' =&gt; 'riak\\_search\\_kv\\_hook', 'fun' =&gt;
&gt; 'precommit'));' instead of like so: '$prop\\_arr = array('precommit' =&gt;
&gt; array(array('mod' =&gt; 'riak\\_search\\_kv\\_hook', 'fun' =&gt; 'precommit')));'.
&gt;
&gt; It's a very subtle difference but the first one creates this:
&gt; '{"props":{"precommit":{"mod":"riak\\_search\\_kv\\_hook","fun":"precommit"}}}'
&gt; while the second creates this
&gt;
&gt; '{"props":{"precommit":[{"mod":"riak\\_search\\_kv\\_hook","fun":"precommit"}]}}'.
&gt; The difference is the square bracket and this threw the whole thing out of
&gt; whack when it wasn't present.
&gt;
&gt; So now everything is working ok.
