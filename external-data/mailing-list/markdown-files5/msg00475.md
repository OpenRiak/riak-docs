&gt; On May 28, 2010, at 11:51 PM, Sam Tingleff wrote:
&gt;
&gt; &gt; I've filed issue #5 on the java client: RiakObject should not present
&gt; &gt; a getValueAsBytes() method
&gt; &gt;
&gt; &gt; The real problem is that if the caller does not use stream(),
&gt; &gt; RiakClient will call getResponseBodyAsString() on the HttpMethod
&gt; &gt; object. This will of course convert from byte[] to String, which is
&gt; &gt; broken behavior for non-string data and is unexpected if the caller
&gt; &gt; uses getValueAsBytes() in RiakObject.
&gt; &gt;
&gt; &gt; IMHO RiakObject should only hold a byte array and perform byte[] to
&gt; &gt; String conversion only on demand and when explicitly asked to do so
&gt; &gt; (if at all).
&gt; &gt;

&gt; packets, if you want to get fancy.
&gt; &gt;&gt;&gt; My suspicion is that they are being modified somewhere in the Java
&gt; stack. Riak is completely agnostic about any binary data you give it.
