2015-03-13 7:08 GMT+05:00 Kota Uenishi :
&gt; Which version of Riak CS are you using? If it's 1.5.3 or later,
&gt; "storage\\_calc\\_timeout" in "riak\\_cs" section should be used instead of
&gt; "mapred\\_timeout" because it won't work [1]. This is undocumented, but
&gt; we should've documented this.

Already used after upgrade to 1.5.3:

{storage\\_calc\\_timeout, 600000}

Value was increased to 900000 now.
Bucket calc errors removed, but all calculation takes too much time
and too much load after cluster extension.

I think, it may be riak issue, not riak-cs.

&gt; Moreover, in CS 1.5.3 or later "mapred\\_timeout" or "timeout" in
&gt; "riakc" section won't work due to the same reason - all names of
&gt; configuration knobs can be found in code [2]. I hope if you don't read
&gt; Erlang, this code might be simple enough to understand what's
&gt; configurable.
&gt;
&gt; [1] 
&gt; https://github.com/basho/riak\\_cs/blob/release/1.5/RELEASE-NOTES.md#additions
&gt; [2] 
&gt; https://github.com/basho/riak\\_cs/blob/release/1.5/src/riak\\_cs\\_config.erl#L418
&gt;
