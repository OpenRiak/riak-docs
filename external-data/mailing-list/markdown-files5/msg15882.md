Are you noticing any Solr errors in the logs? 

For your container instance, you can attempt to clear the AAE trees and force a 
rebuild by removing the entropy directories in `./data/yz\\_anti\\_entropy` and 
running `yz\\_entropy\\_mgr:init([])` via `riak attach`. Or, you can let AAE occur 
naturally (after removing the entropy data) and up the 
concurrency/build\\_limit/tick (using set\\_env). You can see what you’re current 
settings are by calling...

```
riak\\_core\\_util:rpc\\_every\\_member\\_ann(application, get\\_env, [riak\\_kv, 
anti\\_entropy\\_build\\_limit],infinity).
riak\\_core\\_util:rpc\\_every\\_member\\_ann(application, get\\_env, [riak\\_kv, 
anti\\_entropy\\_concurrency],infinity).
riak\\_core\\_util:rpc\\_every\\_member\\_ann(application, get\\_env, [yokozuna, 
anti\\_entropy\\_tick],infinity).
```

… on any of the nodes. Query coverage is R=1, but the values should be 
replicated across.

Thanks.
