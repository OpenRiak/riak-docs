&gt;&gt; Any benefit to using IRiakClient vs RawClient other than what's
&gt;&gt; apparent in the API spec?
&gt;&gt;
&gt;
&gt; RawClient leaves you to deal with all the trade-offs that are consequences of 
&gt; using fault-tolerant, distributed databases like Riak, IRiakClient offers one 
&gt; set of possible well-behaved approaches.
&gt;
&gt; With RawClient, you need to retry a failure, IRiakClient has a default 
&gt; retrier and lets you provide a custom one of your own to handle faults. 
&gt; RawClient will give you an array of Riak Objects and a vector clock where 
&gt; IRiakClient makes it easy to convert the payload to a domain type to simplify 
&gt; sibling resolution. IRiakClient will always fetch before a store and will 
&gt; pass the vector clock for you.
&gt;
&gt; Which you chose depends on what code you want to write and the style you 
&gt; prefer to code in. IRiakClient is there to make it harder for people to shoot 
&gt; themselves in the foot and to incorporate some best practices for being a 
&gt; well behaved Riak client, RawClient is used by IRiakClient, and can certainly 
&gt; be used to build your own interactions with Riak.
&gt;
&gt; I hope that helps explain why there are there 2 layers that you can use, one 
&gt; building on the other.
&gt;
&gt; Please feel free to ask away if I can clear it up any more,
&gt;
&gt; Cheers
