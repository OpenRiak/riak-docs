&gt;&gt; On Fri, Apr 6, 2012 at 3:18 PM, Kresten Krab Thorup 
&gt;&gt; wrote:
&gt;&gt; &gt; It looks like you may have missed specifying the charset when importing
&gt;&gt; &gt; your data; could that be the case?
&gt;&gt; &gt;
&gt;&gt; &gt; You need to specify the charset when importing 8-bit text.  It looks
&gt;&gt; &gt; like your xml is utf-8 encoded, so it should be imported using something
&gt;&gt; &gt; like this:
&gt;&gt; &gt;
&gt;&gt; &gt; curl -H 'Content-Type: text/html;charset=UTF-8' -X PUT @datafile.xml
&gt;&gt; &gt; http://host:port/riak/bucket/key
&gt;&gt; &gt;
&gt;&gt; &gt; The various language clients have different ways of specifying the
&gt;&gt; &gt; charset for a value; so if you imported the xml using some other method you
&gt;&gt; &gt; need to find out where to specify it.
&gt;&gt; &gt;
&gt;&gt; &gt; Perhaps to verify, you can check the result of a curl -v (verbose, print
&gt;&gt; &gt; the headers) for one of your values.  If it does not come back with a
&gt;&gt; &gt; charset=XXX in the Content-Type header, then this is your problem.
