&gt; Sean,
&gt; 
&gt; I did some math based upon the app.config and LOG files. I am guessing that 
&gt; you are starting to thrash your file cache.
&gt; 
&gt; This theory should be easy to prove / disprove. On that one node, change the 
&gt; cache\\_size and max\\_open\\_files to:
&gt; 
&gt; cache\\_size 68435456
&gt; max\\_open\\_files 425
&gt; 
&gt; If I am correct, the node should come up and not cause problems. We are 
&gt; trading block cache space for file cache space. A miss in the file cache is 
&gt; far more costly than a miss in the block cache.
&gt; 
&gt; Let me know how this works for you. It is possible that we might want to 
&gt; talk about raising your block size slightly to reduce file cache overhead.
&gt; 
&gt; Matthew
