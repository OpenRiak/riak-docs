This is caused by a pattern mismatch in your shell. Your left-hand side of
the assignment is

{ok, [{0, [S]}]}

which indicates that the list returned inside the inner tuple is going to
have only a single element, S. If you change it to:

{ok, [{0, S}]}

it will work, assuming S is not already previously bound. S will be a list.

--steve

&gt;&gt; There are two problems with your Mapf function:
&gt;&gt;
&gt;&gt; 1. The body of the function incorrectly uses Object, which is a variable
&gt;&gt; in your shell from command 2, rather than Obj, the incoming parameter to
&gt;&gt; the function. This is the source of the function\\_clause error you're
&gt;&gt; getting.
&gt;&gt;
&gt;&gt; 2. You've stored a binary value in your object, not an Erlang term, so
&gt;&gt; binary\\_to\\_term will fail here with a badarg exception. Change your Mapf
&gt;&gt; function to this instead:
&gt;&gt;
&gt;&gt; Mapf = fun(Obj,\\_,\\_) -&gt; [riak\\_object:get\\_value(Obj)] end.
