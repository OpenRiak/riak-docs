&gt;&gt;&gt; On Mon, Jun 23, 2014 at 4:38 PM, Sean Cribbs  wrote:
&gt;&gt;&gt;
&gt;&gt;&gt;&gt; Hi Alex,
&gt;&gt;&gt;&gt;
&gt;&gt;&gt;&gt; By default, the client assumes new objects use JSON as the encoding.
&gt;&gt;&gt;&gt; The protocol buffers are only used on the wire to encode Riak's protocol.
&gt;&gt;&gt;&gt; If you want to use your own protobuffs messages, I suggest two things:
&gt;&gt;&gt;&gt;
&gt;&gt;&gt;&gt; 1. Decide on a `content\\_type` property for each of your data types and
&gt;&gt;&gt;&gt; be sure to assign that to each key as you create it. I've used
&gt;&gt;&gt;&gt; 'application/pb-person' as an example below.
&gt;&gt;&gt;&gt; 2. Register encoder and decoder functions [1] on the RiakClient object,
&gt;&gt;&gt;&gt; like so:
&gt;&gt;&gt;&gt;
&gt;&gt;&gt;&gt; client.set\\_decoder('application/pb-person', Person.ParseFromString)
&gt;&gt;&gt;&gt; client.set\\_encoder('application/pb-person', lambda x:
&gt;&gt;&gt;&gt; x.SerializeToString())
&gt;&gt;&gt;&gt;
&gt;&gt;&gt;&gt; [1]
&gt;&gt;&gt;&gt; http://riak-python-client.readthedocs.org/en/latest/client.html#serialization
&gt;&gt;&gt;&gt;
&gt;&gt;&gt;&gt; Cheers,
&gt;&gt;&gt;&gt;
