Other tweaks:

- the 2.0 pre-releases have some changes to the networking code that
can help increase, if you're willing to try out an early release.

&gt; for your benchmarks.

&gt; On Fri, Mar 7, 2014 at 10:54 PM, Sean Cribbs  wrote:
&gt;&gt;
&gt;&gt; Note as well that the 'concurrent' setting will \\*not\\* typically increase
&gt;&gt; throughput beyond a certain point (instead you will simply have many workers
&gt;&gt; waiting in the run queue). You are also only driving load from a single
&gt;&gt; machine with limits on its CPU and network throughput. Start by trying a
&gt;&gt; more reasonable concurrency level, like two (2) times the number of CPU
&gt;&gt; cores in the basho\\_bench machine. Also consider running basho\\_bench from
&gt;&gt; multiple machines at once. Consider the line rate of your network as well;
&gt;&gt; in the past I have easily been able to saturate a gigabit network with
&gt;&gt; basho\\_bench.
&gt;&gt;
