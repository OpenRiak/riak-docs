&gt;&gt;&gt;&gt; 
&gt;&gt;&gt;&gt;&gt; It can't be said enough times but the number one thing you can do to 
&gt;&gt;&gt;&gt;&gt; ensure that you are getting true performance (not to mention redundancy) 
&gt;&gt;&gt;&gt;&gt; is to use different physical hardware for each of your nodes. Under no 
&gt;&gt;&gt;&gt;&gt; circumstances should you have more than one VM (one logical node in a 
&gt;&gt;&gt;&gt;&gt; Riak cluster) on the same physical hardware. Also, use multiple 
&gt;&gt;&gt;&gt;&gt; connections/threads/parallelism/whatever on client side and be sure to 
&gt;&gt;&gt;&gt;&gt; hit all the nodes in the cluster haproxy roundrobin-esque when writing to 
&gt;&gt;&gt;&gt;&gt; Riak. Everything else is in the noise.
