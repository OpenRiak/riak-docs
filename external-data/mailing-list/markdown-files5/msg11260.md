Hi Bren,


On Sat, May 11, 2013 at 8:06 PM, Bren Norris  wrote:

&gt; I'm evaluating Riak for an implementation I have planned.
&gt;
&gt; I'd like to know if a single key can be referenced from multiple buckets
&gt; as to create the following scenario:
&gt;
&gt; johnsFriends contains bettysProfile
&gt; davesFriends contains bettysProfile
&gt; betty contains bettysProfile
&gt;
&gt; betty updates profile and all inferences to the profile receive the
&gt; benefit of that update.
&gt;
&gt; Basically, not to dissimilar to an object reference, and if you know LDAP,
&gt; very similar to "Aliases".
&gt;
&gt;
&gt;
This wouldn't be possible in the way you're envisioning it. Keys are unique
to buckets. That said, one way you might do it:

# Store pointers to friends as part of the profile object.

bucket = profile
key = john-profile
value = {name:, }
 {location:, }
 .
 .
 {friends:
 {Name, betty\\_profile}
 {Name, cmeik\\_profile}
 }

This would get problematic when that profile object got too big, but if the
number of friends was bounded (doubtful) or you were OK with writing a new
object after it hit a given size, you could do it this way.

Hope that helps.
