&gt; Hi Chaturi,
&gt;
&gt; $ riak config describe ring\\_size
&gt; Documentation for ring\\_size
&gt; Number of partitions in the cluster (only valid when first
&gt; creating the cluster). Must be a power of 2, minimum 8 and maximum
&gt; 1024.
&gt;
&gt; Valid Values:
&gt; - an integer
&gt; Default Value : 64
&gt; Value not set in /etc/riak/riak.conf
&gt; Internal key : riak\\_core.ring\\_creation\\_size
&gt;
&gt; Cuttlefish configuration schemas are self-describing (via the riak config
&gt; describe command). You can use the name of the internal key in
&gt; advanced.config. If there's nothing else in advanced.config yet, the
&gt; setting looks like
&gt;
&gt; [
&gt; {riak\\_core,
&gt; [
&gt; {ring\\_creation\\_size, 2048}
&gt; ]
&gt; }
&gt; ].
&gt;
&gt; Also see
&gt; http://docs.basho.com/riak/latest/ops/advanced/configs/configuration-files/#Advanced-Configuration
&gt; for more details about using the advanced configuration file.
&gt;
