&gt; On 6 April 2014 15:02, Matthew Von-Maszewski  wrote:
&gt; Edgar,
&gt; 
&gt; In Riak 1.4, there is no advantage to using empty values versus deleting.
&gt; 
&gt; leveldb is a "write once" data store. New data for a given key never 
&gt; physically overwrites old data for the same key. New data "hides" the old 
&gt; data by being in a lower level, and therefore picked first.
&gt; 
&gt; leveldb's compaction operation will remove older key/value pairs only when 
&gt; the newer key/value is pair is part of a compaction involving both new and 
&gt; old. The new and the old key/value pairs must have migrated to adjacent 
&gt; levels through normal compaction operations before leveldb will see them in 
&gt; the same compaction. The migration could take days, weeks, or even months 
&gt; depending upon the size of your entire dataset and the rate of incoming write 
&gt; operations.
&gt; 
&gt; leveldb's "delete" object is exactly the same as your empty JSON object. The 
&gt; delete object simply has one more flag set that allows it to also be removed 
&gt; if and only if there is no chance for an identical key to exist on a higher 
&gt; level.
&gt; 
&gt; I apologize that I cannot give you a more useful answer. 2.0 is on the 
&gt; horizon.
&gt; 
&gt; Matthew
