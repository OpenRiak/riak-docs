2015-03-13 6:45 GMT+05:00 Kazuhiro Suzuki :

&gt; You can change the timeout for a MapReduce job the storage calculation
&gt; uses. Could you try to add riakc section which contains mapred\\_timeout
&gt; into Riak CS's app.config like this ? :

&gt; {riakc, [
&gt; {mapred\\_timeout, 1800000}], %% msec
&gt; },
&gt;

Current config contain:

 {riakc, [ %% increase timeout for LARGE bucket statistic calculation
 {mapred\\_timeout, 3600000}
 %% default mapred\\_call\\_timeout == 60000
 { mapred\\_call\\_timeout, 180000 }
 ]}

It's help to calculate without errors, but does not improve calculation speed.
