I am wondering what's the best value for ring\\_creation\\_size, the default is
64. According to the docs 64 will work for a cluster of no more than 6
nodes (64 /10), and ring\\_creation\\_size of 128 will allow cluster of up to
12 mode. I am wondering what kind of overhead of is associated with a
large ring\\_creation\\_size. Since changing the ring\\_creation\\_size will
result in rebuilding the cluster(destuctive), would a larger value make
more sense and allow scaling by adding more nodes?
