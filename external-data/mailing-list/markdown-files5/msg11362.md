On Mon, Jul 1, 2013 at 8:46 AM, Ryan Zezeski  wrote:

&gt; Riak Users,
&gt;
&gt; Today I'm excited to bring you the 0.7.0 release of Yokozuna. It includes
&gt; some new features such as an upgrade to Solr 4.3.0, isolation of index
&gt; failures, one-to-many index-to-buckets relationship, and map-reduce
&gt; support. There is also a performance improvement in index throughput.
&gt; Along with several bug fixes. See the release notes for more detail.
&gt;
&gt; https://github.com/basho/yokozuna/blob/master/docs/RELEASE\\_NOTES.md#070
&gt;
&gt; Once again I have forgone the EC2 AMI. Only a source package is
&gt; available. You can find instructions for installing on the INSTALL page.
&gt;
&gt; https://github.com/basho/yokozuna/blob/master/docs/INSTALL.md
&gt;
&gt; For those that have been using Yokozuna. The one-to-many change is a
&gt; breaking change. Creating an index no longer implicitly indexes the bucket
&gt; with the same name. Two steps must be performed. FIrst you create the
&gt; index as before. Second you add a bucket property 'yz\\_index' whose value
&gt; is the name of the index you wish to index that bucket under. There is an
&gt; example in the README.
&gt;
&gt; https://github.com/basho/yokozuna#creating-an-index
