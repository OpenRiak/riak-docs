Toby,

I checked on this and you are correct that it is a bug. The query parameters 
for the usage API requests are omitted during the URL processing step. This 
leads to the not\\_requested return you are seeing because when the usage request 
is processed after the URL processing step the indicators that any data was 
requested are no longer there. 

Would you mind filing an issue on the riak\\_cs github repo for this?
