&gt;
&gt;&gt; As of now, the new Java client for Riak v2.0 is available via Maven
&gt;&gt; Central.
&gt;&gt;
&gt;&gt; To use in your project:
&gt;&gt;
&gt;&gt; 
&gt;&gt; 
&gt;&gt; com.basho.riak
&gt;&gt; riak-client
&gt;&gt; 2.0.0
&gt;&gt; 
&gt;&gt; ...
&gt;&gt; 
&gt;&gt;
&gt;&gt; Javadocs are published to: http://basho.github.io/riak-java-client/2.0.0/
&gt;&gt;
&gt;&gt; To get started see:
&gt;&gt;
&gt;&gt; http://basho.github.io/riak-java-client/2.0.0/com/basho/riak/client/api/RiakClient.html
&gt;&gt;
&gt;&gt; First Q: Why did you write a whole new client with a completely different
&gt;&gt; API?!?
&gt;&gt;
&gt;&gt; A: It was time.
&gt;&gt;
&gt;&gt; The new client design addresses a number of issues customers and OSS
&gt;&gt; users have posed over the last two years.
&gt;&gt;
&gt;&gt; First and foremost ... It's built from the ground up to be
&gt;&gt; asynchronous. We're now using Netty in the core of the client. You can
&gt;&gt; now use our client both synchronously or asynchronously.
&gt;&gt;
&gt;&gt; In addition, it's now designed to talk to a Riak cluster, with built
&gt;&gt; in (pluggable) load balancing, node management, etc. These are
&gt;&gt; features users have been asking for, which were near impossible to
&gt;&gt; insert into the old client.
&gt;&gt;
&gt;&gt; Beyond that, our long term maintenance costs are reduced, so we can
&gt;&gt; now provide what we feel is a first rate client going forward.
&gt;&gt;
&gt;&gt; Thank you,
&gt;&gt; - Brian Roach
