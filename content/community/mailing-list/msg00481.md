&gt;&gt; On May 28, 2010, at 11:51 PM, Sam Tingleff wrote:
&gt;&gt; 
&gt;&gt;&gt; I've filed issue #5 on the java client: RiakObject should not present
&gt;&gt;&gt; a getValueAsBytes() method
&gt;&gt;&gt; 
&gt;&gt;&gt; The real problem is that if the caller does not use stream(),
&gt;&gt;&gt; RiakClient will call getResponseBodyAsString() on the HttpMethod
&gt;&gt;&gt; object. This will of course convert from byte[] to String, which is
&gt;&gt;&gt; broken behavior for non-string data and is unexpected if the caller
&gt;&gt;&gt; uses getValueAsBytes() in RiakObject.
&gt;&gt;&gt; 
&gt;&gt;&gt; IMHO RiakObject should only hold a byte array and perform byte[] to
&gt;&gt;&gt; String conversion only on demand and when explicitly asked to do so
&gt;&gt;&gt; (if at all).
