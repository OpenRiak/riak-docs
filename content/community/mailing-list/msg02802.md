Hi Muhammad,
Are you trying to build the riak-java-client from source? It looks like it. I 
guess those errors you have listed are from within eclipse. So, when maven 
downloads the internet, I mean the dependencies, it puts them in a special 
directory on your machine (your local repo). If you have not messed with your 
maven settings at all then your local repo is in your home directory, in a 
folder called .m2 ( ~/.m2/repository )

Eclipse needs to know about the location of your local repo. To tell eclipse 
where your repo is open eclipse preferences and chose Java -&gt; Build Path -&gt; 
Classpath Variables. If there is no variable for M2\\_REPO you need to create it. 
Hit "new" enter M2\\_REPO for name and chose "folder" and browse to 
~/.m2/repository and hit "ok". If M2\\_REPO is already set just hit "Edit" and 
again browse to ~/.m2/repository and hit "ok". Eclipse will ask if it can do a 
full rebuild of your project, say yes and your problems should be over.

In answer to your second question: if you run mvn those jars will be in your 
local repo (~/.m2/repository). Otherwise you will have to find them on their 
respective sites (commons http client, apache commons, commons logging, google 
proto bufs etc.)

I recommend you use maven to get yourself started.

Hope that helps

Russell

&gt; Hi Muhammad,
&gt; The riak java client depends on a couple of external libraries (eg. Apache 
&gt; commons HttpClient, google protobufs). These need to be on your class path 
&gt; when you run your program. The easiest way to do this is to use maven[1]. 
&gt; Using maven means you can simply declare the client as a dependency of your 
&gt; project and maven will take care of downloading the jar (and it's dependent 
&gt; jars). If you are using an IDE (like eclipse) you can then get up and running 
&gt; very quickly indeed.
&gt; 
&gt; Quick start:
&gt; 
&gt; 1. get maven
&gt; 2. run mvn archetype:generate
&gt; 3. edit the pom.xml (that mvn generated) and add the following to 
&gt; dependencies &lt;---- If you already have a project this is all you need to do.
&gt; 
&gt; 
&gt; com.basho.riak
&gt; riak-client
&gt; 0.14.0
&gt; pom
&gt; 
&gt; 
&gt; 4. run mvn clean:install
&gt; 5 . [optional] if you have eclipse installed run mvn eclipse:eclipse to 
&gt; generate project and class path files.
&gt; 6. start hacking
&gt; 
&gt; Of course, if you don't want to get involved with maven you can just grab the 
&gt; following jars and put them on your class path:
&gt; 
&gt; riak-client-0.14.0.jar
&gt; commons-codec-1.2.jar 
&gt; commons-httpclient-3.1.jar
&gt; commons-logging-1.0.4.jar
&gt; protobuf-java-2.3.0.jar
&gt; 
&gt; Have a read of the README at https://github.com/basho/riak-java-client/
&gt; 
&gt; and post back here if you have any trouble.
&gt; 
&gt; Cheers
&gt; 
&gt; Russell
&gt; 
&gt; [1] http://maven.apache.org/download.html (I recommend 2.2.1)
