&gt; &gt; Hi Michael,
&gt; &gt; 
&gt; &gt; 
&gt; &gt; There is no hard coded limit to the number of nodes that can
&gt; &gt; belong to a cluster. Depending on how quickly you are adding
&gt; &gt; nodes to the cluster you may be running into an issue with
&gt; &gt; the ring not converging on a single state.
&gt; &gt; 
&gt; &gt; 
&gt; &gt; Can you try using the "riak-admin ringready" command between
&gt; &gt; each join of a node? This command will return TRUE if the
&gt; &gt; nodes all agree on a particular ring state. You should run
&gt; &gt; this command until it returns TRUE after each join.
&gt; &gt; 
&gt; &gt; 
&gt; &gt; Also, Alexander is correct; you should deploy your nodes
&gt; &gt; with an increased value for the "ring\\_creation\\_size"
&gt; &gt; parameter in app.config. The typical recommendation is to
&gt; &gt; have a minimum of 10 partitions per node and the value
&gt; &gt; should be a power of 2 (64, 128, 256, 512, 1024). For a 63
&gt; &gt; node cluster a value of 1024 would work well.
