&gt;&gt; It's awesome for ad-hoc querying, at least. An example can better explain.
&gt;&gt;
&gt;&gt; Consider this:
&gt;&gt;
&gt;&gt; db.add('users').map('query', '.address .street where
&gt;&gt; .weight:expr(x !.expired').run()
&gt;&gt;
&gt;&gt;
&gt;&gt; as opposed to:
&gt;&gt;
&gt;&gt; db.add('users').map(function(v) {
&gt;&gt; v = Riak.mapValuesJson(v)[0];
&gt;&gt; var result = [];
&gt;&gt; if ((v.weight &lt; 180 || v.exempt) && v.acl && v.acl.state === '1101'
&gt;&gt; && !v.expired) {
&gt;&gt; if (v.address) {
&gt;&gt; result.push(v.address.street);
&gt;&gt; }
&gt;&gt; }
&gt;&gt; return result;
&gt;&gt; }).run()
&gt;&gt;
&gt;&gt;
&gt;&gt; riak-ql is basically adding some query sugar (where, &&) on top of
&gt;&gt; JSONSelect... which you can try it out here:
&gt;&gt; http://jsonselect.org/#tryit
