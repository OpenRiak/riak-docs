I don't totally understand what you're doing in your code, but it looks like
you have the map phase before the link phase which doesn't make sense since
you want the data from the link phase passed on to the map phase, not the
other way around.

On Fri, May 6, 2011 at 9:37 AM, Joshua Hanson wrote:

&gt; In my map/reduce query I would like to keep the input data
&gt; and also data from the link-walking phase.
&gt;
&gt; Here is some sample code:
&gt;
&gt; #insert message
&gt; db.save('messages', 'josh-123', 'secret message', function(err, message) {
&gt; #insert people object with link to message we just inserted
&gt; db.save('people', 'josh', {'profession': 'developer'},
&gt; { links: [{ bucket: 'messages', key: 'josh-123', 'tag': 'message' }]},
&gt; function(err, data) {
&gt; db.add([['people', 'josh']])
&gt; .map({ 'source': 'Riak.mapValuesJson', 'keep': true})
&gt; .link({ 'bucket': 'messages'})
&gt; .run(function(err, data) {
&gt; if (err) return console.log(err);
&gt; console.log(data);
&gt; })
&gt; }
&gt; )
&gt; })
&gt;
&gt; So, if I remove the 'link' phase I get the correct object from the map
&gt; phase
&gt; but if I instead remove the 'map' phase, I get the correct object from link
&gt; phase.
&gt;
&gt; However, having both together does not work. Is it possible to get at both
&gt; the original data and the data from link-walking in the same query?
