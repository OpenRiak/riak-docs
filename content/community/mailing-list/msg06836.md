I'm just saying that to efficiently store and get the data user should have
ability to customize the relationship. Riak has several options to get
related data: secondary indexes, map-reduce etc. I think that the most
important task of mapper is to validate the data and to track these
dependencies. When you have enough meta information about objects storing
these objects is not a big problem.

&gt;&gt;&gt;&gt;&gt; Hi!
&gt;&gt;&gt;&gt;&gt;
&gt;&gt;&gt;&gt;&gt; Two things force me to avoid using Riakkit: the code is GPL licensed
&gt;&gt;&gt;&gt;&gt; and non pep8 compliant. So, I cannot fork it and make changes. The code
&gt;&gt;&gt;&gt;&gt; needs clean up, for example, json module is imported in document.py, but
&gt;&gt;&gt;&gt;&gt; not used. Riakkit has no conflict resolution logic, it's basically 
&gt;&gt;&gt;&gt;&gt; delegate
&gt;&gt;&gt;&gt;&gt; the work to underlying python client. If I'd like to implement my save
&gt;&gt;&gt;&gt;&gt; method I need to copy and rewrite more than 150 lines of code mess. What I
&gt;&gt;&gt;&gt;&gt; think the mapper should do:
&gt;&gt;&gt;&gt;&gt; 1. Serialize the objects to valid json;
&gt;&gt;&gt;&gt;&gt; 2. Deserialize the data back to class instances;
&gt;&gt;&gt;&gt;&gt; 3. Validate the fields data;
&gt;&gt;&gt;&gt;&gt; 4. Track dependencies between models;
&gt;&gt;&gt;&gt;&gt;
&gt;&gt;&gt;&gt;&gt; Simple example using Riakkit-like syntax;
&gt;&gt;&gt;&gt;&gt;
&gt;&gt;&gt;&gt;&gt; class Post(Document):
&gt;&gt;&gt;&gt;&gt;
&gt;&gt;&gt;&gt;&gt; bucket\\_name = 'posts'
&gt;&gt;&gt;&gt;&gt;
&gt;&gt;&gt;&gt;&gt; title = types.StringProperty(required=True)
&gt;&gt;&gt;&gt;&gt; content = types.StringProperty(required=True)
&gt;&gt;&gt;&gt;&gt;
&gt;&gt;&gt;&gt;&gt;
&gt;&gt;&gt;&gt;&gt; class Comment(Document):
&gt;&gt;&gt;&gt;&gt;
&gt;&gt;&gt;&gt;&gt; bucket\\_name = 'comments'
&gt;&gt;&gt;&gt;&gt;
&gt;&gt;&gt;&gt;&gt; text = types.StringProperty(required=True)
&gt;&gt;&gt;&gt;&gt; post = types.IndexReferenceProperty(reference\\_class=Post,
&gt;&gt;&gt;&gt;&gt; collection\\_name='comments')
&gt;&gt;&gt;&gt;&gt;
&gt;&gt;&gt;&gt;&gt; &gt;&gt;&gt; post = Post('title', 'content')
&gt;&gt;&gt;&gt;&gt; &gt;&gt;&gt; comment1 = ('comment1', post=post)
&gt;&gt;&gt;&gt;&gt; &gt;&gt;&gt; comment2 = ('comment2', post=post)
&gt;&gt;&gt;&gt;&gt; &gt;&gt;&gt; post.is\\_valid()
&gt;&gt;&gt;&gt;&gt; True
&gt;&gt;&gt;&gt;&gt; &gt;&gt; post.serialize()
&gt;&gt;&gt;&gt;&gt; {
&gt;&gt;&gt;&gt;&gt; "key": "post\\_key",
&gt;&gt;&gt;&gt;&gt; "title": "title",
&gt;&gt;&gt;&gt;&gt; "content": "content",
&gt;&gt;&gt;&gt;&gt; "comments": [
&gt;&gt;&gt;&gt;&gt; {
&gt;&gt;&gt;&gt;&gt; "key": "comment1\\_key",
&gt;&gt;&gt;&gt;&gt; "text": "comment1"
&gt;&gt;&gt;&gt;&gt; },
&gt;&gt;&gt;&gt;&gt; {
&gt;&gt;&gt;&gt;&gt; "key": "comment2\\_key",
&gt;&gt;&gt;&gt;&gt; "text": "comment2"
&gt;&gt;&gt;&gt;&gt; },
&gt;&gt;&gt;&gt;&gt; ]
&gt;&gt;&gt;&gt;&gt; }
&gt;&gt;&gt;&gt;&gt; &gt;&gt;&gt; post.store()
&gt;&gt;&gt;&gt;&gt; True
&gt;&gt;&gt;&gt;&gt; &gt;&gt;&gt; post = Post.get\\_with\\_key('post\\_key')
&gt;&gt;&gt;&gt;&gt; &gt;&gt;&gt; post.comments
&gt;&gt;&gt;&gt;&gt; ['Post comment1', 'Post comment2']
&gt;&gt;&gt;&gt;&gt; &gt;&gt;&gt; post\\_json\\_data = get\\_json\\_data\\_from\\_request()
&gt;&gt;&gt;&gt;&gt; &gt;&gt;&gt; post = Post.deserialize(post\\_json\\_data)
&gt;&gt;&gt;&gt;&gt; &gt;&gt;&gt; post.title
&gt;&gt;&gt;&gt;&gt; 'title'
&gt;&gt;&gt;&gt;&gt; &gt;&gt;&gt; post.comments
&gt;&gt;&gt;&gt;&gt; ['Post comment1', 'Post comment2']
&gt;&gt;&gt;&gt;&gt;
&gt;&gt;&gt;&gt;&gt; Something like that.

&gt;&gt;&gt;&gt;&gt;&gt; I started writing riakkit approximately 3 months ago and announced it
&gt;&gt;&gt;&gt;&gt;&gt; here. In case you missed that, Riakkit is basically an object mapper for
&gt;&gt;&gt;&gt;&gt;&gt; python and riak, which seemed to be lacking in at the time. Now that 3
&gt;&gt;&gt;&gt;&gt;&gt; months has passed and I have developed this project into much more than
&gt;&gt;&gt;&gt;&gt;&gt; what I had envisioned to begin with. It's currently also being used and
&gt;&gt;&gt;&gt;&gt;&gt; tested in a project of mine (web application) as well and functioning 
&gt;&gt;&gt;&gt;&gt;&gt; quite
&gt;&gt;&gt;&gt;&gt;&gt; nicely. I would say that currently this is the only object mapper that 
&gt;&gt;&gt;&gt;&gt;&gt; can
&gt;&gt;&gt;&gt;&gt;&gt; compete with their counterparts in other languages for Riak in Python,
&gt;&gt;&gt;&gt;&gt;&gt; although a lot of work still needs to be done before it could fully 
&gt;&gt;&gt;&gt;&gt;&gt; compete.
&gt;&gt;&gt;&gt;&gt;&gt;
&gt;&gt;&gt;&gt;&gt;&gt; So today I felt that riakkit is "stable" and "clean" enough to
&gt;&gt;&gt;&gt;&gt;&gt; deserve the first beta release. So I whipped up a website and it's 
&gt;&gt;&gt;&gt;&gt;&gt; released
&gt;&gt;&gt;&gt;&gt;&gt; to pypi. Here's (again), a demo (the full tutorial can be found on the
&gt;&gt;&gt;&gt;&gt;&gt; links below)
&gt;&gt;&gt;&gt;&gt;&gt;
&gt;&gt;&gt;&gt;&gt;&gt; class User(Document):

&gt;&gt;&gt;&gt;&gt;&gt;

&gt;&gt;&gt;&gt;&gt;&gt;

&gt;&gt;&gt;&gt;&gt;&gt; &gt;&gt;&gt; user.save()
